# Leaf-JEPA: Domain-Specific I-JEPA for Plant Disease Detection

## Project Overview

Adapting Meta's I-JEPA (Image Joint-Embedding Predictive Architecture) to agricultural imagery through domain-specific self-supervised pretraining, then evaluating parameter-efficient fine-tuning (PEFT) methods for plant disease classification.

**Two research contributions:**
1. Domain-specific SSL pretraining on PlantVillage to learn disease-aware representations
2. Systematic comparison of LoRA, Adapters, VPT, and BitFit for label-efficient adaptation

**Research questions:** 
Can domain-pretrained I-JEPA outperform generic I-JEPA on a linear probe (RQ1)? 
Which PEFT method achieves the best accuracy/parameter tradeoff (RQ2)? 
Does the best PEFT method match full fine-tuning within 2% at <2% of parameters across all label fractions (RQ3)? 
Does domain pretraining improve cross-domain transfer to PlantDoc (RQ4)? 
Does Leaf-JEPA + PEFT outperform supervised CNNs at ≤10% labels (RQ5)?

## Setup

### 1. Clone and install
```bash
git clone https://github.com/<your-username>/leaf-jepa.git
cd leaf-jepa
pip install -r requirements.txt
```

### 2. Download datasets

Place datasets at the paths below (update `config.py` if your paths differ):
```
data/plantvillage_raw/   # folder-per-class structure
data/plantdoc_raw/       # folder-per-class structure
```

### 3. Run Stage 2 pipeline

Open and run `stage2_notebooks/00_run_all_stage2.ipynb`. This executes notebooks 01–07 in order and produces all required outputs under `outputs/`.

After Stage 2, verify `outputs/quality_verification/integrity_certificate.json` shows `"status": "PASSED"` before proceeding.

### 4. Before running Stage 3

- Copy the computed normalisation stats into `stage3_baselines/config_stage3.py`:
```python
  # from outputs/preprocessing/normalisation_stats.json
  NORM_MEAN = [...]
  NORM_STD  = [...]
```
- Download the I-JEPA checkpoint (ViT-H/14, ImageNet-1K, 300 epochs):
```bash
  wget https://dl.fbaipublicfiles.com/ijepa/IN1K-vit.h.14-300e.pth.tar \
       -P checkpoints/
```
- Update `IJEPA_CHECKPOINT` and `WANDB_ENTITY` in `stage3_baselines/config_stage3.py`.

These must not change after splits are created. Changing any seed or split ratio after Stage 2 invalidates all downstream experiments.

## Backbone

- Architecture: ViT-H/14 (`vit_huge_patch14_224` in timm)
- Source: Meta I-JEPA official checkpoint — ImageNet-1K, 300 epochs
- Embedding dim: 1280 | Patch size: 14 | Parameters: 632M
- Loading key: `target_encoder` (EMA encoder — higher quality representations)

## Primary Evaluation Metric

**Macro-F1** (not accuracy). PlantVillage has class imbalance; accuracy rewards majority-class bias. All results reported as mean ± std across 3 seeds.
